现在网络环境越来越差，五一期间又出现了大规模 DNS 污染的事情。现在用的几家机场面临的生存压力比较大，也逐渐在涨价。所以平时就在寻找一些小众的代理软件，这样被和谐的几率较低，同时最好是设置简单，毕竟不是专做这个东西肯定没有太多的精力去研究。偶然在 Youtube 上看到熊教学一个介绍 Brook 的视频，查了一些资料感觉这款软件还不错，就拿谷歌云来试一试。

这款软件的优点是：配置简单，速度快，小众比较低调。
缺点：不如 SSR 那么成熟，客户端比较简陋。当然这都能接受。

服务端配置

其实这款代理软件在逗比根据地有详细的介绍，逗比大神也发出了一键安装的脚本，我这里只是再多啰嗦一遍了。

我的系统版本：Ubuntu 16.04.4 LTS (GNU/Linux 4.13.0-1011-gcp x86_64)

下载脚本安装

wget -N --no-check-certificate https://softs.fun/Bash/brook.sh && chmod +x brook.sh 
sudo bash brook.sh

Note: Root 下运行脚本，否则可能会出现管理脚本下载失败的情况。

分别按照提示输入 1 后填入端口和密码，协议直接默认使用 Brook 协议即可。

完成后会显示以下信息。

[信息] Brook 启动中...
[信息] Brook 启动成功 !

Brook 用户配置：
————————————————
 地址	: X.X.X.X
 端口	: 2333
 密码	: doub.io
 协议	: Brook(新版)
 Brook 链接 : brook://default%XXXXXXXXXXXXXX

[注意] Brook链接 仅适用于Windows系统的 Brook Tools客户端（https://doub.io/dbrj-7/）。

Note： 记得把 VPS 防火墙的相应端口开放，这样才能正常访问。
